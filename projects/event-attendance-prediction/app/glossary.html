What each info means

A plain-language guide to every number in the estimation tool and the results dashboard. The model learns attendance rates from the EVE study data.

RSVP responses

Confirmed
Guests who answered Going. Most likely to attend — but far from all do, especially at big events.
Maybe
Guests who answered Interested / Maybe. A minority actually show up.
Declined
Guests who answered No. Treated as zero attendance.
No reply
Invited guests who never responded. Individually unlikely to come, but at large events they are the biggest group, so they still add up.
Total invited
The sum of all four responses — everyone who was invited.

Event size

Size class
Derived automatically from total invited. Conversion (who actually shows up) drops sharply as events grow, so the model uses a different rate per class.
Micro · ≤ 200 Small · 201–1500 Medium · 1501–5500 Large · ≥ 5501
Type
Physical (guests must be at a place) or Virtual (online). Recorded with the event.

External effects

External effect
Something outside the invite that changes turnout — a decisive match, rain, a competing event. You give it a ± %; the estimate is multiplied by it. e.g. −20% for a championship final the same night.

The estimate

Estimate
Predicted number of people present. Computed as Σ (count × rate for that response & size), then adjusted by any external effects.
Likely range
The estimate ± the average error seen for that size class (±10% micro … ±50% medium). Smaller events are predicted more tightly.
Where the number comes from
The breakdown bars: how many of the estimated attendees come from each response type. Confirmed usually dominates.

History

Actual attendance
The real turnout you enter after the event. This is what lets the model be recalibrated.
Error
| actual − estimate | — how far the prediction was, in people.
Export CSV
Downloads your saved events so they can be added to the dataset and the model refit.

Dashboard metrics

Conversion
Σ attendance ÷ Σ confirmed for a size class — the share of confirmations that turned into presence.
Predicted vs. actual
Each event as a point; on the dashed diagonal, the prediction was perfect. Above = over-predicted, below = under-predicted.
MAE
Mean Absolute Error — the average miss in people. Lower is better.
MAPE
Mean Absolute Percentage Error — the average miss as a percent of actual turnout.
Mean accuracy
100 − MAPE — a single at-a-glance score. ~76% on the current data.

The model

Learned rates
Attendance probability per response type and size, estimated from the study data — not guessed by an operator (that was the old EVE's weak point).
In-sample · v1
Fitted and measured on the same small dataset (20 events). It's an honest first calibration; adding your real events (via Export CSV) improves it.

Rule of thumb: raw RSVP counts overstate turnout. The tool exists to translate responses into a realistic estimate — and to get better as you log what actually happened.
