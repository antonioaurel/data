Pull Request Review Flow

The current Claude Orchestrator to Codex workflow for pull request review, discussion logging, operator visibility, and merge decision support.

Document typeEngineering Workflow
StatusDraft
Primary reviewerCodex
Final decisionUser

Collaboration Principle

Codex is the reviewer, not a subordinate of Claude. Claude implementation agents own implementation work on the pull request. The Claude Orchestrator owns the queue and handoffs. Codex owns independent review judgment.

When they disagree, both agents should try to converge on a practical agreement based on evidence from the code, tests, project constraints, and user intent.

If no agreement is possible, the user is the final decision maker. The preferred handoff is a concise summary of the disagreement, the risks of each option, and the recommended next step.

Flow Diagram

Claude implementation agents open PRs and report them to the Orchestrator. The Orchestrator triggers the local Codex review bridge, Codex posts review comments, and Claude iterates on fixes until the PR is ready for the user decision.

1

Claude creates PR

Implementation branch is opened as a GitHub pull request.

2

Claude reports PR

The implementation agent sends the PR number to the Orchestrator queue.

3

Orchestrator runs bridge

python3 tools/review_pr_with_codex_cli.py <PR> --notify-claude --notify-from-pr

4

Read PR metadata

The bridge uses gh pr view for title, URL, base, and head.

5

Fetch PR branch

The PR head is fetched into an isolated remote ref.

6

Create worktree

A temporary checkout is created under /private/tmp/codex-pr-*.

7

Run Codex review

codex review --base origin/<base> reviews the PR.

8

Post review

Codex findings are posted as a GitHub PR comment.

9

Post completion

Completion summary includes discussion points and merge guidance.

10

Clean up

The temporary worktree is removed after the run.

11

Orchestrator monitors

Terminal output and .pr-review-logs/dashboard.md show status.

12

Claude addresses

Claude reads review comments and updates the PR branch, then the Orchestrator reruns review.

13

User decides

The user decides whether to merge or accept remaining risk after Codex is satisfied.

Orchestrator Contract

The Orchestrator is the queue manager. It receives PR numbers from Claude implementation agents, runs Codex review, waits for fixes, reruns review after each fix commit, and stops only when Codex reports no blocking findings or the user explicitly accepts the remaining risk. 

python3 tools/review_pr_with_codex_cli.py <PR_NUMBER> \
  --notify-claude \
  --notify-from-pr

Operator Visibility

Terminal

The script prints when the review starts, when comments are posted, and when the review finishes.

Dashboard

.pr-review-logs/dashboard.md shows RUNNING, COMPLETED, or FAILED.

PR Comments

The pull request remains the canonical audit log for review findings, discussion, and decisions.

Current Limitation

The Orchestrator still needs to run the review bridge after a Claude implementation agent opens or updates a PR. A future GitHub Action could remove that requirement by triggering the review on pull_request: opened, synchronize, and reopened.
